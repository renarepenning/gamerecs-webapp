Additional information
  • Data collection tool & methods

    • The primary source of data is from the video game database IGDB. IGDB provided an ideal basis for our data collection, as it has 346,742 game entries along with ample information accompanying each game such as genres, release dates, and the studio that produced it. We were able to web scrape in Python from the IGDB API and obtain JSON files of all of this data. For human data, we created and distributed an online survey that asks users for their favorite indie games, as well as other games they like and profile questions such as their favorite gaming genres and their age.

  • Algorithm development methodology

    • The algorithm was developed in multiple phases, the first model used the basic relationship between a user input and 5 independent variables that contain qualitative arrays. To quantify these variables, our algorithm creates a score based on the similarities between the user imputed gameâ€™s characteristics for each variable, and the characteristics of each datapoint in that variable. To do this, we take the conjunction of each data point in that specific variable, and divide it by the disjunction. The lengths of these corresponding outputs create a quantitative score that is summed over all independent variables to create a master score. The second phase of the algorithm was adding in Indie weights. The larger game studios have more qualitative data, and thus generally receive higher recommendations. To be more inclusive, we included an Indie dummy variable that will augment the score of indie games, boosting their recommendation. After that, to backtest our algorithm, we asked the user to rate our recommendation. In addition, our algorithm randomly assigns weights to each independent variable that change with each iteration. This randomness allows the algorithm to vary its recommendations, and develop a specific dependent variable for statistical analysis. Note that the randomness function to assign the weights can be skewed to target variables more or less, and creates the quantifiable dataset needed to find optimal weighting.

  • UX/UI tools

    • Our website is a Django web app hosted on Heroku. We used Bootstrap to further aid in stylistic choices. The website includes a user model that allows the user to create a profile. Within that profile, the user can retrieve new recommendations by filling out a form. Each recommendation that the user gathers is stored, along with their rating, in the database. This allows the algorithm to access the ratings and also allows the user to view their old recommendations and ratings at any time via the user page.
